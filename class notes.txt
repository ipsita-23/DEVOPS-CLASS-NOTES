EXECUTABLE COMMANDS to be used with docker run :
	-it       : runs the container in interactive mode allowing u to interact with the container in the interactive mode
	-d        : runs the container in the background, detached mode
	--name    : specifies a name for the container
	--rm      : automatically removes the container when it exits
	-p        : maps a host port to a container port
	-e/ --env : sets an environment variable inside the container
	-v        : mounts a host volume inside the container
	examples: docker run -it httpd , docker run -d httpd
docker ps :shows all running containers
docker ps -a :shows all the containers
docker run httpd echo "Hello, World!"                             //randomly takes any container, gives name by itself
docker run --name my-container httpd echo "Hello, World!"   //running a container with a predefined name , here: my-container

Docker provides 3 types of data storage:
1. Volumes-
   is persistent, is managed by docker, used in databases, production
2. Bind Mounts-
   is persistent, not managed by docker, used for development
3. tmpfs-
   not persistent, managed by docker in memory, used for secrets, cache


Docker Volumes- Independent of container lifecycle, can be shared by multiple containers, managed fully by a docker

Without volumes-> data is stored inside the container, and on deleting the container, data is lost
With volumes-> data is stored on the host machine, containers can be recreated without losing data

docker volume create <volume name>
docker volume ls
docker volume inspect mydata
docker run -d -v <volumename>:<path where to store in image> --name <containername><imagename> 
eg:
docker volume create mydata
docker volume ls
docker volume inspect mydata
docker run -dit -v mydata:/app/data --name cont1 ubuntu  //-v: for volume , mydata:/app/data ->container address
docker exec -it cont1 bash
